# last-seen-network

Publish current machine network state to `~/setup/<hostname>/LAST_SEEN_NETWORK`.
Syncthing propagates this file to the fleet. Consumer skills (e.g. `fleet-ssh-tunnel`)
use it for peer discovery.
This is deliberately LAN-bound — if you need cross-NAT fleet reachability, use
Tailscale/Headscale instead.

## Output file

Path: `~/setup/<hostname>/LAST_SEEN_NETWORK`

```
# Auto-generated by last-seen-network.timer + NM dispatcher (85-last-seen-network)
# Never edit by hand.
#
# Fields:
#   lan        LAN IPv4 in CIDR (e.g. 10.4.5.123/24), empty if no route
#   gateway    Default gateway IPv4, empty if no route
#   iface      Primary route interface. Naming reveals medium: en*/eth* = ethernet, wl* = wifi
#   ssid       SSID of any currently-associated wifi AP (even when ethernet is primary),
#              empty if no wifi association
#   network    Matched network-policy network name, or '*'
#   zone       Matched network-policy zone name, or '*'
#   ip_source  Which public-IP endpoint answered:
#              mullvad | ifconfig.co | icanhazip | ipify | none
#   ip, country, country_code, city, longitude, latitude,
#   mullvad_exit_ip, mullvad_exit_ip_hostname, mullvad_server_type,
#   mullvad_server_provider, blacklisted, organization
#              Fields returned by public-IP endpoints; populated to whatever depth
#              the responding endpoint provides. Empty if not available.
#   updated    ISO 8601 UTC timestamp of this write
lan=10.4.5.123/24
gateway=10.4.0.1
iface=wlp0s20f3
ssid=alpine
network=alpine
zone=home
ip_source=mullvad
...
updated=2026-07-04T22:15:00Z
```

## Requirements

- **network-policy** — dispatcher must write current `network`, `zone`, and `iface`
  to `/run/network-policy/current` (tmpfs) on every `up` event. This skill includes
  an `85-last-seen-network` dispatcher hook and a systemd Path watcher that triggers
  on changes to that file.
- **Syncthing** — must be active and syncing the `domovoy-setup` folder so
  `LAST_SEEN_NETWORK` reaches peers.

## Triggers

Three triggers (all domovoy user units):

1. **`last-seen-network.timer`** — fires every 5 minutes after boot
2. **`last-seen-network.path`** — watches `/run/network-policy/current`;
   fires on any change (network event)
3. **`85-last-seen-network`** — NM dispatcher hook (optional safety net):
   directly invokes the service on `up`/`dhcp4-change`/`dhcp6-change`/`connectivity-change`

The timer provides steady-state coverage (every 5 min to refresh geo data);
the Path unit provides instant updates on network transitions.

## Script logic

1. Determine primary interface via default route
2. Get LAN IPv4 CIDR and default gateway on that interface
3. Enumerate wifi interfaces for SSID (even when ethernet is primary — AirTag use case)
4. Read `network` and `zone` from `/run/network-policy/current` (populated by network-policy dispatcher)
5. Query public IP with fallback chain (5s timeout each):
   - `https://am.i.mullvad.net/json`
   - `https://ifconfig.co/json`
   - `https://icanhazip.com/`
   - `https://api.ipify.org?format=json`
6. Write `LAST_SEEN_NETWORK.tmp`, atomic `mv` to final path
7. If no default route: skip the write entirely (no network → nothing to publish;
   Syncthing can't propagate anyway)

## No-route behavior

When there is no default route (fully offline), the script does not touch
`LAST_SEEN_NETWORK`. The file retains its last successful state. Downstream
consumers see the `updated=` timestamp go stale and treat the publisher as
unreachable. No `error=` field needed — stale timestamp is the signal.

## Installation

```bash
# Copy script
cp ~/Public/domovoy-skills/last-seen-network/last-seen-network ~/.local/bin/
chmod +x ~/.local/bin/last-seen-network

# Copy systemd user units
cp ~/Public/domovoy-skills/last-seen-network/last-seen-network.{service,timer,path} \
    ~/.config/systemd/user/

# Enable timer + path watcher
systemctl --user daemon-reload
systemctl --user enable --now last-seen-network.timer last-seen-network.path

# First run
last-seen-network

# Verify
cat ~/setup/$(hostname)/LAST_SEEN_NETWORK
```

### NM dispatcher (root, optional safety net)

```bash
sudo cp ~/Public/domovoy-skills/last-seen-network/85-last-seen-network \
    /etc/NetworkManager/dispatcher.d/
sudo chmod 755 /etc/NetworkManager/dispatcher.d/85-last-seen-network
```

## Dependencies

- `curl` (for public-IP queries)
- `iw` or `nmcli` (for SSID detection)
- `ip` (iproute2) — for route and address queries
- `date` (GNU coreutils) — for ISO 8601 timestamps
